# Лабораторная работа №2: Коллекция объектов CharacterCollection

## Коллекция — это структура данных, предназначенная для хранения и управления группой однотипных объектов. В объектно-ориентированном программировании коллекции предоставляют удобный интерфейс для работы с множеством объектов, скрывая детали внутренней реализации.

### Преимущества использования коллекций
- **Инкапсуляция** — скрывает внутреннюю структуру хранения данных
- **Контроль доступа** — позволяет проверять корректность добавляемых объектов
- **Удобство использования** — предоставляет понятные методы для работы с группой объектов
- **Расширяемость** — легко добавлять новые методы работы с коллекцией

## Предметная область

Используется тот же вариант, что и в лабораторной работе №1 — "Игровые персонажи". Основная сущность — **Персонаж (Character)**.

### Коллекция CharacterCollection управляет группой персонажей, позволяя:

- Добавлять новых персонажей в группу
- Удалять персонажей из группы
- Искать персонажей по различным критериям
- Фильтровать персонажей по параметрам
- Получать подмножества персонажей (например, здоровых или сильных)
- Сортировать персонажей по разным характеристикам

## Коллекция CharacterCollection

### Назначение коллекции CharacterCollection

Коллекция `CharacterCollection` предназначена для:

- Хранения объектов класса `Character` (из лабораторной работы №1)
- Управления группой персонажей (добавление, удаление, поиск)
- Предоставления удобного доступа к элементам коллекции
- Реализации операций фильтрации и сортировки
- Обеспечения целостности данных (проверка типов, защита от дубликатов)

## Атрибуты и методы

**Метод** — это функция, принадлежащая классу, которая определяет поведение объектов этого класса. Методы позволяют взаимодействовать с объектом, изменять его состояние или получать информацию о нем.

### Основные методы управления

| Метод | Описание |
|-------|----------|
| `add(character)` | Добавляет персонажа в коллекцию |
| `remove(character)` | Удаляет персонажа из коллекции |
| `remove_at(index)` | Удаляет персонажа по индексу |
| `get_all()` | Получает список всех персонажей |

#### Особенности метода `add()`:
- Проверяет, что добавляемый объект — действительно персонаж
- Проверяет, нет ли такого персонажа в коллекции (по уникальному имени)
- Добавляет персонажа в конец списка

#### Особенности метода `remove()`:
- Находит и удаляет указанного персонажа из коллекции
- Вызывает ошибку, если персонажа нет в коллекции

#### Особенности метода `remove_at()`:
- Удаляет персонажа, находящегося на позиции `index`
- Возвращает удаленного персонажа
- Проверяет, что индекс существует

### Поиск

Поиск — это процесс нахождения элементов коллекции, удовлетворяющих определенному условию. В коллекции `CharacterCollection` реализовано несколько методов поиска.

| Метод | Описание |
|-------|----------|
| `find_by_name(name)` | Находит персонажа по имени |
| `find_by_health_range(min, max)` | Находит персонажей в диапазоне здоровья |
| `find_by_power_range(min, max)` | Находит персонажей в диапазоне силы |


#### Особенности методов поиска:
- `find_by_name()` возвращает одного персонажа или `None`
- Методы с `_range()` возвращают список персонажей
- Поиск по имени регистронезависимый

### Фильтрация

**Фильтрация** — это процесс отбора элементов коллекции, удовлетворяющих определенному условию. Методы фильтрации создают новую коллекцию, не изменяя исходную.

| Метод | Описание |
|-------|----------|
| `get_healthy(min_health)` | Возвращает новую коллекцию здоровых персонажей |
| `get_strong(min_power)` | Возвращает новую коллекцию сильных персонажей |

#### Особенности методов фильтрации:
- Создают новую коллекцию (не изменяют исходную)
- Включают только персонажей, удовлетворяющих условию
- Сохраняют все методы исходной коллекции
